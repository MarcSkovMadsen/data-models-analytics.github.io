The best toolset for interactive analytics?

  • ti 07 maj 2019
  • analytics
  • #interactive analytics

I'm creating interactive analytics and data visualizations for my users all the time. And I'm a bit frustrated as my current toolsets are not efficient.

Currently I have too supported toolsets to choose from.

  • BI Tools like Power BI and Tableau
  • Frontend web development using Angular

Both are impressive technologies with a lot of pros. But both keeps frustrating me over and over again because of their cons.

I need something in between and I'm currently seeing Jupyter Notebooks for prototyping and Plotly Dash for production as the best "in between" solution.

You can take a look at impressive dashboards using Dash in the Dash Gallery

Why Tableau ends up frustrating me.

In Tableau you can do rapid prototyping and release to production using drag and drop. But

  • since you don't get code files it often ends up being difficult to develop and maintain in the long run.
  • you often end up just not being able to do exactly what is needed or have to use some unmaintainable hack to solve the problem.
  • Tableau is also costly in it self and yet another technology to master, so often the business analyst, data engineer and data scientist cannot work with the same technology.

Why Angular ends up frustrating me.

With Angular you have code and all the tools at hand for tests, documentation, refactoring, writing advanced models etc. But

  • you end up having two stacks of tools. One for the backend and one for the front end.
  • you have to spend a lot of time integrating and keeping the backend and front end in sync.
  • you probably develop some of the same business logic in the two stacks.
  • you have to spend time on customizing Angular for data visualization as it not a dedicated solution for that.
  • your team now probably needs additional backend and front end resources.

The setup is now very complex and the development/ maintenance cycle is very slow.

We just needed a dashboard for internal use and not a complex web application for thousands of users!

I need something in between

I need something in between with the possibility of

  • Presenting data in a efficient, appealing and interactive way.
  • Ability to integrate into web sites to combine with other technologies like Angular when it's really needed.
  • Keep the technology stack as simple as possible as i'm not working in large teams.
  • Developing in Python mainly as this
    • is the common language for people working with data, models and analytics.
    • has a vast eco-system for data analysis and visualization.
    • A fast develop-test-build-release-maintain cycle
    • can be refactored, structured into modules, documented, tested etc.

I really think we could speed up our development time and do valuable stuff based on this.

What would be my preferred toolset?

My preferred toolset and workflow would be

  • Python for data extraction, transformation and loading.
  • Jupyter Notebooks for rapid prototyping of Dashboards
  • Dash for production Dashboards.

When I really need something more advanced for specific use cases I could add in

  • Django as backend when multiple dashboards, security, admin stuff, forms, data download etc. is needed.
  • Angular when advanced, custom front end development is really needed.

Why Jupyter Notebooks?

A Jupyter Notebook is the common tool for data exploration and prototyping among data engineers and data scientists.

Why Dash?

With Dash you can develop impressive, interactive, web based dashboards in Python only. Dash creates all the frontend html, css and javascript for you. Dash is focused on rapid development on interactive visualizations.

In Dash you just need to write Python functions to extract, transform and load data and they will be called automatically whenever the user interacts with your dashboard.

Dash can be simple because you can develop an advanced dashboard in just one python file if you like.

Dash is a web application and can easily be deployed.

Dash can be combined with for example Django backend and Angular frontend if your requirements are to build a complex application where you need REST APIs, Logins/ Security, SPAs for some of the parts.

For more examples take a look at the impressive dashboards in the Dash Gallery . There are links to the code as well.
